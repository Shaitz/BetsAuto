Description

Event flow

Main Flow

  1. User selects a Change Username and/or Password button
  2. System sets username and Password fields as editable
  3. User changes username and/or Password
  4. User clicks Accept
  5. System sets username and Password fields as non editable and saves their value

Alternativ flow

  • 5.A If Username and Password match with those of an already existing User, changes are not made
  • 5.B If the Username and the Password haven't recieved any change, changes are not saved
