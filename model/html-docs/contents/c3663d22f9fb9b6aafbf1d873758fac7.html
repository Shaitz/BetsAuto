Description

Event flow

Main Flow

  1. User selects a question
  2. System displays the BetQuestionGUI with the info of selected question
  3. User enters the answer and the amount of money he wants to bet
  4. User clicks Bet button
  5. System displays success message

Alternative flow

  • X.A User can quit at any moment, END
  • 5.B If there is no no money on his wallet, display error message
  • 5.C If there is an empty field, display error message
