Bet&Ruin :: Use Case Diagram :: Login/Register

Description

Login/Register

Register flow

  1. User enters username and password
  2. User clicks Register
  3. System adds new user to the database
  4. System closes LoginRegisterGUI

Register alternative

  • X.A User can quit at any moment, END
  • 3.B if user already exists in the database, display error message
  • 3.C if one or more fields are empty, dispaly error message

Login flow

  1. User enters username and password
  2. User clicks Login
  3. System retrieves user from the database
  4. System closes LoginRegisterGUI

Login alternative

  • X.A User can quit at any moment, END
  • 3.B if user dosn't exist in the database, display error message
  • 3.C if one or more fields are empty, dispaly error message

Precondition

  • No logged user
